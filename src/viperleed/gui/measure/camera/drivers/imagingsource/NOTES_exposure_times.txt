CONCLUSIONS:
* continuous mode yields more accurate times per frame than triggered mode,
  especially at large exposure times (>150ms / frame).
* However, continuous mode reproducibly takes significantly longer to deliver
  the first frame, at least as long as the exposure time, and up to 150ms more.
* This means that it is worth using continuous mode only when n_frames is large
  enough to compensate for the extra times:
  
  dt_trigger = (E + d_tr_1)*n_frames + d_tr_0
  dt_cont = (E + d_co_1)*n_frames + E + d_co_0
  
  Trig_minus_Cont = (d_tr_1 - d_co_1)*n_frames + d_tr_0 - E - d_co_0
  
  It is thus convenient to use triggered if:
     (d_tr_1 - d_co_1)*n_frames < E + d_co_0 - d_tr_0
  i.e.,
     if n_frames < (E + d_co_0 - d_tr_0) / (d_tr_1 - d_co_1)
  
  d_tr_1 - d_co_1 < 5ms/frame below 300ms exposure; 40ms/frame @2000ms
  
--> With DMK 33GX265, it looks like TRIGGERED BURST is always more convenient
    in terms of total time required (at least if <= ~30--50 frames). However,
	for very short exposures there is a limit on the number of frames that can
	be delivered with one trigger burst.
